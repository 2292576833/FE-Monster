# Components Usage Guide

这个目录保存可复用的界面组件与配套样式。每次执行 UI 相关任务时，先理解用户真正要达成的效果，再检查这里是否已有合适组件；有合适组件时应优先调用或复用其样式，而不是重新手写一套。

## 执行顺序

1. 先理解用户意图  
   明确用户要的是功能、布局、视觉材质、动画效果，还是某个具体交互。不要只按字面改一个元素，要判断它在当前界面中的作用。

2. 先查组件目录  
   开始写代码前查看 `components/`，确认是否已有可复用组件或 CSS。若用户点名某个文件，例如 `GlassSurface.jsx` 或 `GlassSurface.css`，优先阅读并调用它。

3. 优先调用现有组件  
   如果当前界面是 React/Vite 页面，直接 import 对应 `.jsx` 与 `.css`。如果当前界面是原生 HTML 页面，也应优先复用对应 `.css`、结构约定或视觉参数。

4. 只在必要时新写  
   现有组件无法满足任务、或接入成本明显高于收益时，再补充新实现。新实现应保持和现有组件的命名、圆角、动效、颜色与交互习惯一致。

5. 完成后验证  
   检查目标页面是否真的加载了组件或样式，确认没有 console 错误、布局重叠、文字溢出、按钮不可点、动画空白等问题。

## 组件索引

- `GlassSurface.jsx` / `GlassSurface.css`  
  玻璃材质容器。适合播放栏、浮层、控制面板、黑灰玻璃 UI、需要 backdrop blur 的小型面板。

- `ElectricBorder.jsx` / `ElectricBorder.css`  
  电流边框。适合当前播放、选中态、主要按钮、重点面板的强调，但不要滥用。

- `LineWaves.jsx` / `LineWaves.css`  
  线性波形背景。适合音乐视觉、背景能量场、音频响应氛围。

- `MagicBento.jsx` / `MagicBento.css`  
  Bento 卡片网格。适合功能入口、模块集合、可扫描的信息卡组。

- `ScrollStack.jsx` / `ScrollStack.css`  
  堆叠滚动卡片。适合分段内容、展示型滚动体验。

- `AnimatedList.jsx` / `AnimatedList.css`  
  动画列表。适合队列、搜索结果、歌单条目等列表内容。

- `AnimatedContent.jsx`  
  内容进入动画。适合轻量显隐、状态切换、局部内容出现。

- `FadeContent.jsx`  
  淡入内容。适合低干扰的内容过渡。

- `GradientText.jsx` / `GradientText.css`  
  渐变文字组件。仅在明确需要品牌展示或视觉标题时使用，产品控件标签不要默认使用。

- `Cubes.jsx` / `Cubes.css`  
  方块互动视觉。适合背景互动或视觉实验，不适合作为常规表单/播放控件。

## 调用原则

- 用户点名的组件或样式文件优先级最高。
- 组件能解决 70% 以上需求时，优先接入后微调。
- 组件只提供视觉材质时，也可以只调用 CSS。
- 不要为了炫技把所有组件都堆上去；组件服务于用户意图。
- 当前任务如果只需要一个小控件，保持小尺寸、低噪音、清晰可点。
