# FE Monster 更新日志

本文记录 FE Monster 客户端与 FE Monster 社区服务器的主要功能更新。
维护范围包括：

- 客户端项目：`E:\FE moster`
- 社区服务器项目：`E:\FE moster server`

## 2026-07-06

### DIY 与视觉界面

- 将 DIY 面板里的“文字预设”提升为一级页签，现在与“预设”“壁纸模式”处于同一层级。
- DIY 页签结构调整为：
  - `预设`：场景预设、低频状态、魔方强度。
  - `文字预设`：3D 歌词、流动文字、歌词亮度、跳动速度。
  - `壁纸模式`：导入壁纸、实时壁纸、壁纸适配模式。
- 新增 `diyTextModeButton` 与 `diyTextPage`，避免文字预设混在场景预设里，侧边栏层级更清楚。
- 更新 DIY 样式规则，三段页签等宽显示，新文字页继承原 DIY 页的滚动、内边距和隐藏规则。

### 搜索、收藏与播放体验

- 搜索栏已支持音乐搜索结果列表，结果项按播放页歌曲栏风格展示。
- 搜索结果支持收藏入口，底部播放栏也保留收藏按钮。
- 收藏逻辑支持选择要收藏到的歌单或收藏目录。
- 搜索结果列表可滚动，同时可以隐藏显式滚动条，减少视觉干扰。
- 针对播放页选择歌曲时的滚动卡顿做过优化，减少列表滚动和当前歌曲状态刷新之间的互相影响。
- 顶部搜索栏保持居中，在小窗口下会压缩按钮与搜索栏尺寸，尽量避免重叠与换行。

### 黑色玻璃 UI 与窗口适配

- 顶部搜索栏、底部播放栏和主要控制表面统一为黑色玻璃质感。
- 下拉内容不纳入黑色玻璃材质，避免选项层级和正文面板混在一起。
- 底部播放栏默认会自动下滑隐藏，指针接近或交互时再出现。
- 小窗模式下顶部按钮保持在顶部，按钮和搜索栏会按可用宽度自适应。
- DIY 侧边栏在小窗口下做了适配，按钮区域可正常点按。

### 手势操作与相机源

- 设置页增加手势操作开关。
- 开启手势操作时，客户端会调用 Python 手势脚本并尝试打开摄像头。
- 手势脚本使用 OpenCV、MediaPipe、PyAutoGUI，支持：
  - 食指移动控制鼠标。
  - 拇指与食指捏合模拟点击。
  - 四指并拢上下移动触发滚动。
  - 五指捏合和张开用于缩小、放大。
  - 特定手势触发快捷键。
  - 稳定时长机制降低误触。
- 设置页增加“摄像头 / 相机”独立切换按钮。
- 相机模式会扩大设备扫描范围，并优先匹配 `canon`、`eos`、`webcam utility` 等名称，方便调用佳能相机或 EOS Webcam Utility。
- 手势服务会把摄像头打开状态、脚本缺失、Python 缺失、依赖缺失、相机不可用等状态反馈到界面。

### 社区客户端功能

- 社区卡片支持自适应窗口大小。
- 社区卡片支持折叠，折叠后只保留头像、徽章、FE ID 和名称等关键信息。
- 徽章名称可以在徽章旁边直接显示。
- 点击社区头像可打开个人资料窗口。
- 个人资料窗口支持 3D 悬浮效果，可拖拽顶部移动位置，可拖拽底部旋转。
- 个人资料支持填写自我描述，长度限制由客户端与服务端共同约束。
- 个人资料顶部加入“社区集体”切换按钮。
- “社区集体”页会显示同区域约 5-10 公里范围内的社区账号，并可发起添加好友。
- 消息窗口缩小尺寸，支持拖拽顶部移动、拖拽底部旋转。
- 收到好友消息时，左侧空白区域会出现消息气泡。
- 消息气泡支持关闭，也支持在气泡里直接回复。
- 社区页增加消息免打扰按钮，开启后后续消息不再弹出气泡。
- 好友经常听歌提示气泡以黑金质感显示好友头像、名称和“喜欢听”，短暂展示后自动淡出。

### 一起听歌与社区实时同步

- 一起听歌支持邀请、同意、拒绝、退出。
- 客户端会上报当前播放歌曲、播放位置、时长、播放状态和可播放状态。
- 社区服务器会把一起听会话中的歌曲变化发布为实时事件，其他成员收到后同步选中音乐并播放。
- 对 VIP 或不可播放歌曲场景，客户端与服务器会保留 `playable`、`duration`、`position` 等字段，方便同步时按可播放一方的状态降级处理。
- 一起听歌面板支持退出会话，服务器会发布 `listen.left` 事件给会话成员。
- 一起听歌保留连麦信令通道，可通过社区服务器中转 WebRTC 信令。

### 社区服务器与局域网连接

- 社区服务器默认监听 `0.0.0.0:3020`，局域网内客户端可通过服务器电脑的局域网 IP 连接。
- 启动脚本会打印本机可用访问地址，优先使用类似 `http://192.168.x.x:3020` 的局域网地址。
- 客户端通过 `/health` 检测社区服务器，只有返回 `ok: true` 且服务名为 `fe-monster-community` 时才认为连接成功。
- 服务器连接失败时，客户端会明确显示连接失败，不再假装在线。
- 客户端登录社区后会自动注册 FE ID、用户名、头像、平台账号、电脑 ID、客户端版本和最后在线状态。
- 服务器记录客户端 IP、User-Agent、在线状态、心跳、听歌状态和实时事件连接数。
- 服务器控制台会实时输出注册、心跳、听歌、消息、点赞、一起听、徽章、更新等事件。

### FE ID 与电脑 ID

- FE ID 规则固定为 `10000000` 到 `99999999` 之间的 8 位随机数字。
- 服务器分配 FE ID 时会检查重复，确保同一服务器内不重复。
- 客户端会读取电脑 ID，优先使用官方闭源社区模块生成的设备指纹；没有闭源模块时会回退为本地安装 ID。
- 社区账号会绑定电脑 ID。服务器下发徽章、修改社区 ID 等操作会校验 FE ID 与电脑 ID 是否匹配。
- 服务器支持通过电脑 ID 修改对应账号的社区 FE ID，新 ID 仍必须落在 8 位数字范围内。

### 社区服务器图形后台

- 社区服务器提供图形管理后台，不再只依赖命令行窗口。
- 后台会自动记录所有已登录客户端 ID，并显示：
  - 已记录客户端数量。
  - 在线客户端数量。
  - 实时连接数。
  - 后台操作数量。
  - FE ID、昵称、IP、平台账号、电脑 ID、客户端版本、好友数、消息数、最近在线时间。
- 后台支持按 ID、昵称、IP、平台账号、电脑 ID、设备名等信息搜索客户端。
- 后台支持向指定 FE ID 下发徽章，并要求目标 FE ID 已绑定电脑 ID。
- 后台支持清除手动徽章，恢复自动认证。
- 后台支持推送新客户端版本，填写版本号、下载地址、更新说明，并可设置自动安装策略。
- 服务端推送新版本后，在线客户端会收到 `update.available` 事件并弹出更新窗口。
- 客户端更新窗口显示新版本号、更新说明和进度；点击立即更新后会下载安装包并启动隐藏安装流程。

### 社区安全与授权边界

- 开源客户端主体使用 MIT License。
- 社区服务端、官方闭源社区模块、签名实现、设备指纹实现和安全实现使用 Proprietary License。
- 开源代码只保留公开接口和 UI 调用，不放置社区签名密钥、加密盐值、最终设备指纹算法或防篡改核心逻辑。
- 闭源社区模块通过公开接口提供：
  - 完整性校验。
  - 设备指纹生成。
  - 动态请求签名。
- 社区服务器支持：
  - HTTPS / mTLS 配置。
  - AppKey + HMAC-SHA256 动态签名。
  - 时间戳与 Nonce 防重放。
  - 设备 ID 与账号绑定。
  - 白名单参数校验。
  - FE ID 格式校验。
  - 每秒请求限流。
  - 异常 IP 切换与攻击特征风控。

## 2026-07-05

### 社区服务器独立化

- 客户端和社区服务器拆分运行。
- 客户端不再自动启动社区服务器，社区服务器由 `E:\FE moster server` 单独管理。
- 服务器启动脚本会检测端口占用；如果 `3020` 已经运行，会提示“server is already running”并打印可连接地址。
- 提供 `set-community-server-url.ps1`，用于把客户端社区地址切换到局域网服务器。

### 实时事件通道

- 新增社区事件流 `/api/community/events`。
- 客户端登录社区后会自动连接事件流。
- 服务器通过事件流实时推送：
  - 好友添加。
  - 在线状态变化。
  - 私聊消息。
  - 点赞。
  - 个人资料更新。
  - 一起听邀请、同意、拒绝、退出、同步。
  - 连麦信令。
  - 徽章下发。
  - 版本更新。

### 社区基础能力

- 新增好友添加接口，按 8 位 FE ID 查找和添加好友。
- 新增好友列表与在线状态刷新。
- 新增私聊消息接口，支持最近会话读取与发送。
- 新增点赞接口，点赞数会参与自动认证。
- 新增听歌时长上报接口，听歌时长会参与自动认证。
- 新增个人资料描述保存接口。
- 新增附近社区账号接口，按区域和 5-10 公里半径返回社区用户。

## 2026-07-01

### Java 客户端基础迁移

- 项目迁移为 Java 17 本地服务 + 静态 Web 客户端结构。
- 主入口为 `src/main/java/com/femonster/FeMonsterJavaApp.java`。
- 默认本地服务端口为 `3000`，端口占用时会自动尝试后续端口。
- 支持 `FE_MONSTER_PORT` 指定端口。
- 支持本地客户端窗口启动，优先使用 Edge / Chrome app 模式。
- 静态托管 `web/` 与 `components/`。

### 播放器与音乐平台

- 新增播放器状态与控制 API：
  - `/api/player/state`
  - `/api/player/play`
  - `/api/player/pause`
  - `/api/player/previous`
  - `/api/player/next`
  - `/api/player/seek`
  - `/api/player/volume`
  - `/api/player/load`
  - `/api/player/queue`
- 新增搜索、封面、歌曲链接和网易云代理接口。
- 播放状态保存到 `data/player-state.json`。
- 支持网易云登录状态持久化。
- 预留 QQ 音乐和酷狗音乐接入，具体可用性取决于本地第三方音乐 API 服务。

### Visual Bridge 与黑色舞台

- 新增 Visual Bridge 状态接口：
  - `/api/visual-bridge/health`
  - `/api/visual-bridge/state`
- Visual Bridge 会根据播放状态生成能量、低频、节拍、颜色和队列数据。
- 前端收敛为沉浸式黑色舞台。
- 底部透明玻璃播放栏保留上一首、播放/暂停、下一首、进度、音量、封面和歌曲信息。
- 新增 3D 立方体预设，为后续更多视觉场景和沙盒模式打基础。
