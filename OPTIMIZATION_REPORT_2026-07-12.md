# FE Monster 客户端优化审计与执行报告

日期：2026-07-12
范围：Java 本地服务、Web 客户端、沙盒模式、DIY 预设、社区创作市场、Three.js/Canvas 渲染与预设数据

## 1. 执行结论

本轮完成了“读取现状 → 建立基线 → UI/预设/性能审计 → 实施 → 构建 → 桌面与 390px 窄屏验收”的闭环。

最明显的性能收益来自静态资源传输层：核心首屏资源由 1,537,959 字节原始传输降到 332,745 字节 gzip，下降 78.4%。GLTFLoader 的 96,550 字节原始文件移出首屏，仅在第一次加载 Blender 组件时请求。

界面方面，沙盒布局原本已经具备较好的视觉基础。本轮重点解决了社区市场横向溢出、DIY 同名预设难以区分、预设缺少成品缩略图、移动端资料面板遮挡和密集按钮过度挂载光效的问题。

## 3. 预设与组件数据审计

- 服务端预设：13 个，共 51,616 字节。
- 服务端组件：14 个，共 42,124 字节。
- 无损坏 JSON、无错误 schema、无缺失 GLB 地址。
- 存在 6 个同名“我的音乐场景”。它们 ID 不同，属于有效但难以辨认的数据，未擅自删除或合并。
- 最大预设约 41 KB，包含 37 个 `fallbackSceneItems`。服务端保留完整回退数据；客户端 localStorage 与草稿缓存现在会剔除 `files` 和 `fallbackSceneItems`，避免重复序列化。
- DIY 页现在按更新时间倒序，并显示来源、组件数、日期与 Blender PNG 缩略图；同名预设可以被区分。

## 4. 已执行优化

### P1：静态资源传输

- Java 静态资源支持 gzip。
- HTML 保持 no-store，确保入口及时更新。
- 带 `v=` 的 CSS/JS 使用一年 `immutable` 缓存。
- 其他静态资源使用一天缓存并附带 ETag。
- 条件请求实测返回 HTTP 304。
- 大型二进制资源改为流式输出，不再统一 `Files.readAllBytes`。

### P1：运行时渲染与轮询

- 主粒子球不再无条件 60fps 运行，按设备档位帧预算调度。
- 沙盒打开或文档隐藏时停止主粒子循环。
- 返回前台或退出沙盒后自动恢复。
- 原生音频 50ms 采样与 Visual Bridge 在文档隐藏时暂停。
- Blender GLTFLoader 改为首次需要时动态加载。

### P1：输入与草稿性能

- 沙盒表单滑杆/数字输入不再每次事件立即 JSON.stringify 全场景。
- 草稿写入合并为 180ms debounce。
- 关闭沙盒、切后台、关闭页面时立即 flush。
- 本地组件、预设和草稿缓存剔除不参与客户端渲染的重复资产清单与 fallback 场景。

### P1：按钮特效成本

- 原逻辑会给页面全部 175 个按钮创建 React 根节点。
- 现在仅顶部导航、播放控制、沙盒主操作等 20 个关键按钮保留光效。
- 市场、预设、组件列表使用轻量 CSS hover/focus。
- 光效模块改为浏览器空闲阶段加载；低性能设备和 reduced-motion 模式跳过。

### P2：UI、响应式和可访问性

- 社区创作市场卡片不再产生横向滚动。
- 市场增加资源类型标签、加载骨架和真实资产预览支持。
- DIY 自定义预设增加高清缩略图、来源、组件数和日期。
- 移动端社区资料页增加遮罩，阻止与背景社区卡片同时交互。
- 社区页签补齐 ARIA tab/tabpanel 语义。
- 沙盒、市场、DIY 和主操作增加统一键盘焦点环。
- 频谱、歌词进度和更新进度从 width 动画改为 transform/clip-path，避免布局抖动。
- 歌词热点文字从渐变文字改为清晰的动态实色。
- 启动按钮强制等待由约 2.18 秒缩短为 720ms；降级兜底由 2.6 秒缩短为 1.2 秒。

## 5. 验证结果

- Java 构建成功，许可证边界检查通过。
- `app.js` 与按钮光效脚本语法检查通过。
- 浏览器控制台无 error/warn。
- 首屏 `load` 实测约 134ms（本地开发环境）。
- 进入按钮在 load 后 900ms 内可用。
- 初始页面不包含 GLTFLoader；进入含 Blender 组件的沙盒后动态脚本数量变为 1，模型正常导入。
- 市场列表：`scrollWidth === clientWidth`，横向溢出为 false。
- 390×844：社区面板底部位于视口内，市场无横向溢出。
- DIY：7 个自定义预设全部生成缩略图容器，其中 2 个 Blender 预设使用真实 PNG。
- 条件静态资源请求返回 304。

## 6. 仍建议继续推进的更新

### P1：拆分前端单体文件

当前 `app.js` 约 620 KB、`styles.css` 约 258 KB。建议按功能拆成：

1. `sandbox-runtime.js/css`
2. `community-market.js/css`
3. `playback-visuals.js/css`
4. `diy-presets.js/css`
5. `core-player.js/css`

优先动态加载沙盒和市场模块。拆分前需先补充浏览器级回归脚本，避免移动代码时破坏共享状态。

### P1：消除首屏 CDN 运行时依赖

启动 Lightfall 仍从 esm.sh 加载 React、ReactDOM 和 OGL。建议将所需版本固定并迁移到本地 vendor，或者改成无 React 的小型 WebGL 启动效果。这样可以避免离线或网络波动导致启动背景失败。

### P2：建立完整设计 token

把社区金色、沙盒青色、错误红、成功绿、文字层级、表面层级统一成 primitive → semantic → component 三层 token。优先替换重复最多的 `rgba(236, 190, 95, …)` 和 `rgba(131, 228, 255, …)`，不要一次重写全部 CSS。

### P2：预设管理能力

- 增加重命名入口。
- 保存同名预设时给出“覆盖 / 另存为”选择。
- 支持按来源、更新时间、2D/3D/粒子筛选。
- 服务端将大型 `fallbackSceneItems` 独立成引用资产，减少市场与预设 JSON 复制。

### P2：图像资源

`rain-glass-blender-bg.png` 约 3.1 MB。建议输出 1920/1280 两档 WebP 或 AVIF，并根据视口和渲染档位选择。原始 PNG 可保留作为高质量回退。

### P2：长期性能监控

建议增加开发模式性能面板，采集：

- 实际 FPS 与长任务次数。
- Canvas/WebGL 分辨率和粒子数量。
- API 请求频率与失败率。
- GLB 加载时间、三角形数量和纹理显存估算。
- 低性能档触发次数。
