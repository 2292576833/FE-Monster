# FE Monster 使用说明

## 启动应用

1. 推荐安装 Java 26；程序仍兼容 Java 17 或更新版本。
2. 在项目根目录运行：

```powershell
.\run.cmd
```

应用会先编译 Java 代码，再启动本地客户端窗口。控制台会显示实际访问地址，例如：

```text
URL: http://127.0.0.1:3000/
```

如果默认端口被占用，应用会自动尝试后续端口。也可以手动指定：

```powershell
$Env:FE_MONSTER_PORT = "3001"
.\run.cmd
```

只启动 Web 服务、不打开本地客户端：

```powershell
.\run.cmd --server
```

## 音乐平台 API

FE Monster 负责界面和本地 Java 代理，二维码登录、搜索、歌单、歌曲链接需要对应的第三方音乐 API 服务先运行。

默认地址如下：

| 平台 | 环境变量 | 默认地址 |
| --- | --- | --- |
| 网易云 | `FE_NETEASE_BASE_URL` | `http://127.0.0.1:3010` |
| QQ 音乐 | `FE_QQ_BASE_URL` | `http://127.0.0.1:3011` |
| 酷狗音乐 | `FE_KUGOU_BASE_URL` | `http://127.0.0.1:3012` |
| 汽水音乐 | `FE_QISHUI_BASE_URL` | `http://127.0.0.1:3013` |

启动 FE Monster 前可以这样配置：

```powershell
$Env:FE_NETEASE_BASE_URL = "http://127.0.0.1:3010"
$Env:FE_QQ_BASE_URL = "http://127.0.0.1:3011"
$Env:FE_KUGOU_BASE_URL = "http://127.0.0.1:3012"
$Env:FE_QISHUI_BASE_URL = "http://127.0.0.1:3013"
.\run.cmd
```

所有平台配置统一保存在独立文件：

```text
data/music-api/providers.json
```

主界面会先启动，音乐 API 在后台探测和拉起；某个平台不可用时不会阻塞或关闭 FE Monster。内置服务和导入服务只允许使用本机回环地址及明确端口。

### 在登录页导入 API

登录弹窗上方的“导入 API”支持两种文件：

- `.json` / `.feapi`：只导入平台名称、本机端口和能力声明，不执行代码，最大 64 KB。
- `.zip`：导入可在本机启动的 API 包，最大 25 MB。ZIP 会运行本地代码，必须在弹窗中明确确认，并且只应导入可信来源。

普通配置示例：

```json
{
  "schema": "fe-monster.music-apis/v1",
  "providers": [
    {
      "id": "qishui",
      "label": "汽水音乐",
      "appName": "汽水音乐 App",
      "baseUrl": "http://127.0.0.1:3013",
      "healthPath": "/health",
      "enabled": true,
      "loginQr": false
    }
  ]
}
```

ZIP 根目录必须包含且只能包含一个 `music-api-package.json`（也兼容 `fe-music-api.json`）：

```json
{
  "schema": "fe-monster.music-api-package/v1",
  "id": "qishui",
  "label": "汽水音乐",
  "appName": "汽水音乐 App",
  "baseUrl": "http://127.0.0.1:3013",
  "healthPath": "/health",
  "autostart": true,
  "loginQr": false,
  "launcher": {
    "runtime": "node",
    "entry": "server.cjs",
    "args": []
  }
}
```

启动器可使用 `node`、`java`、`powershell` 或 `npm`，入口必须位于 ZIP 包内，并以前台常驻进程方式运行。FE Monster 会限制路径、文件数、展开体积和 Node/Java 内存，并在退出或重复导入时回收其托管进程。

可导入的平台 ID 固定为 `netease`、`qq`、`kugou`、`qishui`。通用适配器至少识别 `/search` 与 `/song/url`；如果包支持扫码登录，再声明 `"loginQr": true` 并实现 `/login/qr/key`、`/login/qr/create`、`/login/qr/check`。未声明扫码能力的包仍可搜索和播放，登录页不会把它误报为二维码故障。

第三方 API 参考：

- 网易云：`NeteaseCloudMusicApi`，二维码登录通常使用 `/login/qr/key`、`/login/qr/create`、`/login/qr/check`。
- QQ 音乐：需要选择支持二维码登录端点的 QQ 音乐 API 服务；部分开源 QQ Music API 项目只支持搜索/播放，不实现登录。
- 酷狗音乐：需要选择支持登录、歌单、搜索、歌曲链接的酷狗 API 服务。
- 汽水音乐：已提供独立平台适配槽，可从登录页导入本机 API 配置或可信 ZIP 包。

## 登录使用

1. 点击右上角登录按钮。
2. 在二维码登录弹窗里选择 `网易云`、`QQ音乐`、`酷狗音乐` 或已导入的 `汽水音乐`。
3. 使用对应 App 扫码确认。
4. 登录成功后，主界面会自动读取用户歌单并显示歌单卡片。
5. 点击歌单卡片，会打开悬浮歌单架；点击歌曲即可播放。

网易云登录状态会保存在：

```text
data/netease-auth.json
```

退出重进后会自动尝试恢复网易云账号。QQ 音乐和酷狗音乐是否能保留登录，取决于所接入的第三方 API 服务是否保存 cookie/session。

## 二维码加载不出来

先确认 FE Monster 接入到了正确的第三方 API 地址：

```powershell
Invoke-WebRequest -UseBasicParsing "http://127.0.0.1:3000/api/providers" | Select-Object -ExpandProperty Content
```

再检查对应平台服务：

```powershell
Invoke-WebRequest -UseBasicParsing "http://127.0.0.1:3000/api/netease/login/qr/key" | Select-Object -ExpandProperty Content
Invoke-WebRequest -UseBasicParsing "http://127.0.0.1:3000/api/qq/login/qr/key" | Select-Object -ExpandProperty Content
Invoke-WebRequest -UseBasicParsing "http://127.0.0.1:3000/api/kugou/login/qr/key" | Select-Object -ExpandProperty Content
```

如果返回类似：

```json
{"ok":false,"provider":"qq","label":"QQ音乐","baseUrl":"http://127.0.0.1:3011","error":"QQ音乐 API unavailable ..."}
```

说明 FE Monster 正常工作，但对应第三方 API 没启动、端口不对，或服务没有实现二维码登录接口。

解决顺序：

1. 重新运行 `.\run.cmd`。当前版本会自动清理旧 FE Monster Java 进程，并按 `data/music-api/providers.json` 在后台启动已配置的本机 API。
2. 如果使用外部服务，确认平台端口与 `FE_NETEASE_BASE_URL`、`FE_QQ_BASE_URL`、`FE_KUGOU_BASE_URL`、`FE_QISHUI_BASE_URL` 一致。
3. 也可以在登录页重新导入对应 JSON 配置或可信 ZIP 包。
4. 确认第三方服务支持二维码登录接口，并且 `/login/qr/create` 能返回二维码图片字段，例如 `qrimg`、`qrImg`、`qrcode` 或 `image`。
5. 重新打开登录弹窗，或点击弹窗里的刷新按钮。

## 常用接口

```text
GET /api/providers
GET /api/music-apis
GET /api/music-apis/status?provider=qishui
GET /api/login/status?provider=netease
GET /api/netease/login/qr/key
GET /api/qq/login/qr/key
GET /api/kugou/login/qr/key
GET /api/qishui/login/qr/key
GET /api/search?q=关键词&provider=netease
GET /api/netease/user/playlists
GET /api/qq/user/playlists
GET /api/kugou/user/playlists
```

如果当前实际端口不是 `3000`，把命令里的端口替换成控制台显示的端口。
