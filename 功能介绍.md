# FE Monster 功能介绍

FE Monster 是一个面向 Windows 桌面的沉浸式音乐播放器。它优先使用 Java 26 启动本地服务，并兼容 Java 17+，用 Web UI 呈现黑色玻璃音乐舞台，并通过独立社区服务器提供好友、消息、一起听歌、徽章、更新推送等社区能力。

## 一、播放器基础功能

- 本地启动：运行 `run.cmd` 后启动 Java 服务和本地客户端窗口。
- 本地端口：默认 `3000`，被占用时会自动尝试后续端口，也可用 `FE_MONSTER_PORT` 指定。
- 播放控制：支持播放、暂停、上一首、下一首、拖动进度、调整音量、播放队列。
- 播放状态持久化：当前播放状态保存到 `data/player-state.json`。
- 歌曲搜索：搜索栏可搜索歌曲，显示歌曲名称、歌手、封面和收藏入口。
- 歌单读取：登录音乐平台后读取用户歌单，并在播放页中展示歌曲列表。
- 多平台接口：客户端内置网易云、QQ 音乐、酷狗音乐的代理入口；实际登录、歌单、歌曲链接能力取决于本地对应音乐 API 服务是否支持。
- 收藏：搜索结果和底部播放栏都可以收藏歌曲，收藏时可选择目标歌单或收藏目录。

## 二、黑色玻璃舞台

- 主界面以音乐舞台为中心，播放栏、搜索栏和关键控制区使用黑色玻璃风格。
- 底部播放栏默认自动下滑隐藏，交互时再浮现，减少遮挡。
- 顶部搜索栏保持居中，小窗时会自动压缩搜索栏和按钮，避免重叠。
- 播放页歌曲列表做了滚动优化，减轻选择歌曲时的卡顿感。
- 视觉状态会跟随播放、低频、节拍和当前歌曲信息变化。

## 三、DIY 视觉系统

DIY 侧边栏现在分为三个同层级入口：

| 页签 | 作用 |
| --- | --- |
| 预设 | 管理场景预设，如无场景、动感魔方、Sonic Terrain，并调整低频与魔方强度。 |
| 文字预设 | 管理歌词文字效果，如 3D 歌词、流动文字，并调整歌词亮度与跳动速度。 |
| 壁纸模式 | 管理导入壁纸、实时壁纸和壁纸适配方式。 |

这种层级把“场景”“文字”“壁纸”分开，后续继续添加预设时不会把控制项堆在同一个面板里。

## 四、手势操作

设置页提供手势操作开关。开启后，客户端会调用 Python 脚本并打开摄像头或相机。

已接入能力：

- OpenCV 读取摄像头画面。
- MediaPipe 识别手部关键点。
- PyAutoGUI 执行鼠标移动、点击、滚动、缩放和快捷键。
- 食指指尖控制鼠标移动。
- 拇指和食指捏合触发点击。
- 四指并拢上下移动触发滚动。
- 五指捏合/张开触发缩小/放大。
- 稳定时长机制降低误触。
- 摄像头 / 相机独立切换，方便在普通摄像头和佳能相机之间切换。
- 相机模式会优先匹配 Canon、EOS、Webcam Utility 等设备名。

## 五、社区功能重点介绍

社区功能由独立社区服务器承载。客户端只负责展示和交互，社区服务器负责身份、在线状态、好友关系、消息、一起听歌、徽章、更新推送和安全校验。

### 1. 社区身份：FE ID + 电脑 ID

- 每个社区账号会获得一个 8 位 FE ID。
- FE ID 范围固定为 `10000000` 到 `99999999`。
- FE ID 由服务器随机生成并检查重复。
- 客户端会读取电脑 ID 并上报到服务器。
- 有官方闭源社区模块时，电脑 ID 来自闭源模块生成的设备指纹。
- 没有闭源模块时，客户端会回退为本地安装 ID。
- 服务端下发徽章、修改 FE ID、推送配置等敏感操作必须校验 FE ID 与电脑 ID 绑定关系。

### 2. 社区卡片

社区卡片显示当前社区身份：

- 头像。
- 昵称。
- 音乐平台账号信息。
- FE ID。
- 徽章与徽章名称。
- 在线好友数量。
- 消息与免打扰入口。
- 好友添加输入框。

社区卡片支持窗口自适应和折叠。折叠后只保留头像、徽章、FE ID 和名称，让小窗口下仍然能看到身份信息。

### 3. 好友系统

- 可以输入 8 位好友 FE ID 添加好友。
- 服务端只会在同一社区服务器内查找好友 ID。
- 好友必须已经登录并注册到这台社区服务器，否则会提示“该社区服务器上未找到好友 ID”。
- 好友列表显示在线状态、当前听歌状态、徽章和点赞状态。
- 好友添加成功后，服务器会通过实时事件通知双方刷新状态。

### 4. 消息系统

- 支持好友私聊。
- 消息长度限制为 500 字以内。
- 消息窗口尺寸更小，适合悬浮在播放界面上。
- 可拖拽窗口顶部移动位置。
- 可拖拽窗口底部旋转，形成 3D 悬浮效果。
- 收到新消息时，左侧空白处会出现消息气泡。
- 消息气泡可关闭，也可直接回复。
- 社区页提供消息免打扰按钮，开启后不再弹出新消息气泡。

### 5. 个人资料与社区集体

- 点击社区头像可打开个人资料窗口。
- 个人资料窗口支持移动与旋转。
- 用户可以填写自己的描述，让其他社区成员认识自己。
- 顶部提供“个人资料”和“社区集体”切换。
- “社区集体”页会显示同区域约 5-10 公里内的社区账号。
- 附近账号可直接添加为好友。

说明：当前附近逻辑由社区服务器根据客户端地址推导区域，并在 5-10 公里范围内做稳定排序；它不是精确 GPS 定位。

### 6. 点赞、听歌时长与认证

- 好友之间可以点赞。
- 客户端会上报听歌时长和当前播放歌曲。
- 服务器根据听歌时长和点赞数自动生成认证：
  - 活跃乐友。
  - 认证乐友。
  - 深度乐友。
- 服务端手动徽章优先级高于自动认证。
- 手动徽章可在服务器后台下发或清除。

### 7. 一起听歌

一起听歌用于让好友在同一首歌上保持播放同步。

支持流程：

- 向在线好友发起一起听邀请。
- 对方可同意或拒绝。
- 同意后创建一起听会话。
- 任一成员切换歌曲、播放/暂停或进度变化时，服务器会发布同步事件。
- 其他成员收到同步事件后尝试选中同一首歌并播放。
- 任一成员可退出一起听，服务器会通知会话成员。
- 连麦信令可通过社区服务器中转，为语音或 WebRTC 能力预留通道。

遇到 VIP 或不可播放歌曲时，客户端会上报 `playable`、`duration`、`position` 等字段。同步时可以按可播放的一方状态降级，避免无法播放的一方卡住整个会话。

### 8. 好友常听提示

- 客户端会统计好友听歌数据。
- 当某首歌平均总时长达到约 2 小时，可判定为“爱听”。
- 播放栏上方会出现黑金质感提示气泡。
- 气泡展示好友头像、好友名称和“喜欢听”，约 2 秒后淡出。

### 9. 社区实时事件

社区服务器通过 `/api/community/events` 推送实时事件。已覆盖：

- `presence.updated`：好友上线、离线、听歌状态变化。
- `friend.added`：好友添加。
- `message.sent`：私聊消息。
- `like.added`：点赞。
- `profile.updated`：个人资料更新。
- `listen.invite`：一起听邀请。
- `listen.responded`：一起听同意或拒绝。
- `listen.sync`：一起听歌曲与播放状态同步。
- `listen.left`：退出一起听。
- `call.signal`：连麦信令。
- `badge.assigned`：服务端下发徽章。
- `badge.cleared`：清除手动徽章。
- `update.available`：服务器推送新版本。
- `community-id.changed`：服务器通过电脑 ID 修改 FE ID。

这些事件会实时反馈到服务器控制台，也会触发客户端对应 UI 刷新。

## 六、社区服务器与后台

社区服务器位于 `E:\FE moster server`，默认监听：

```text
0.0.0.0:3020
```

局域网客户端应连接服务器电脑的局域网 IP，例如：

```text
http://192.168.x.x:3020
```

后台能力：

- 图形化服务器 UI。
- `/admin` Web 管理后台。
- 自动记录所有登录客户端。
- 显示在线数、连接数、客户端资料、电脑 ID、版本号、IP、平台账号。
- 搜索客户端。
- 下发徽章。
- 清除手动徽章。
- 通过电脑 ID 修改 FE ID。
- 推送新客户端版本。
- 查看后台操作记录。

## 七、客户端更新

服务器端可发布新客户端版本：

- 版本号。
- 下载地址。
- 更新说明。
- 文件大小。
- SHA-256。
- 是否允许自动安装。

在线客户端收到更新事件后会弹出更新窗口，显示：

- 新客户端版本号。
- 更新说明。
- 更新进度。
- “立即更新”和“稍后”按钮。

用户点击立即更新后，客户端会下载更新包并启动安装脚本。`.exe` 安装包会以隐藏窗口方式运行，并把进度写入 `data/update-progress`。

## 八、安全与授权

FE Monster 当前采用开源客户端 + 闭源社区敏感模块的边界：

- 开源客户端主体：MIT License。
- 社区服务端、闭源社区模块、签名、设备指纹、安全实现：Proprietary License。

安全设计重点：

- 开源部分只调用公开接口，不包含签名密钥和设备指纹盐值。
- 闭源模块负责设备指纹、动态签名和完整性校验。
- 服务端支持 AppKey + HMAC-SHA256 动态签名。
- 请求包含时间戳和 Nonce，服务端限制 5 分钟窗口并缓存 Nonce 防重放。
- 服务端支持 HTTPS 与 mTLS 配置。
- 服务端不信任客户端传入的 user_id，敏感操作会重新校验 FE ID、电脑 ID 和资源权限。
- 所有社区写请求走参数白名单、长度限制、FE ID 格式校验和限流。
- 异常 IP 切换、攻击特征和高频请求会触发风控。

## 九、常用入口

客户端：

```powershell
.\run.cmd
```

社区服务器：

```powershell
E:\FE moster server\run.cmd
```

配置客户端连接社区服务器：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "E:\FE moster\scripts\set-community-server-url.ps1" -Url "http://服务器局域网IP:3020"
```

服务器后台：

```text
http://服务器局域网IP:3020/admin
```
